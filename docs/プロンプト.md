# AI筋トレメニュー生成ツール — Docker環境構築

---

## 指示

以下の仕様に従い、Next.jsアプリをDocker Composeで動かす開発環境を
ゼロから構築してください。

---

## アプリ概要

- **アプリ名**: AI筋トレメニュー生成ツール
- **フレームワーク**: Next.js 15（App Router）
- **言語**: TypeScript
- **スタイリング**: Tailwind CSS
- **AI**: Google Gemini API（gemini-2.5-flash）
- **ホスティング想定**: Vercel（開発環境のみDockerを使用）

### 2. docker-compose.yml

以下の仕様で作成すること：

- サービス名: `app`
- ビルド: `./Dockerfile`
- ポート: `3000:3000`
- ボリューム:
  - `.:/app`（ホットリロードのため）
  - `/app/node_modules`（コンテナ内のnode_modulesを保護）
  - `/app/.next`（ビルドキャッシュを保護）
- 環境変数: `.env.local` から読み込む（`env_file`を使用）
- コマンド: `npm run dev`
- `stdin_open: true` と `tty: true` を設定（コンテナを起動し続けるため）

---

### 3. Dockerfile

以下の仕様で作成すること：

- ベースイメージ: `node:20-alpine`
- 作業ディレクトリ: `/app`
- `package.json` と `package-lock.json` をコピーして `npm install` を実行
- アプリのソースコードをコピー
- ポート `3000` を公開
- デフォルトコマンド: `npm run dev`
- **マルチステージビルドは不要**（開発環境のみ）

### 5. app/page.tsx（選択フォーム）

以下の選択項目をボタンUIで実装すること：

| 項目 | 選択肢 | 選択方式 |
|---|---|---|
| 目標 | 筋肉をつける・脂肪を燃やす・体力をつける・姿勢を改善する | 単一選択 |
| 鍛えたい部位 | 全身・胸・背中・腕・腹筋・脚・肩 | 複数選択可 |
| 経験レベル | 初心者・中級者・上級者 | 単一選択 |
| トレーニング時間 | 15分・30分・60分・90分 | 単一選択 |
| 使用できる器具 | 自重のみ・ダンベル・バーベル・ジム全設備 | 単一選択 |

実装要件：

- `"use client"` を先頭に付ける
- 選択状態は `useState` で管理する
- 選択されたボタンはインディゴ系（`border-indigo-400 bg-indigo-50 text-indigo-700`）で強調する
- 送信時は `/api/generate` に POST する
- 結果は `sessionStorage` に保存して `/result` ページへ `router.push` する
- ローディング中はボタンテキストを「AIがメニューを作成中...」に変更し `disabled` にする
- 画面下部に「AIに送信されるプロンプト（自動生成）」のプレビューを表示する

プロンプトのテンプレート（`buildPrompt` 関数として実装）：

```
あなたはプロのパーソナルトレーナーです。以下の条件に合わせた筋トレメニューを作成してください。

【条件】
- 目標: {goal}
- 鍛えたい部位: {parts}（複数の場合は「・」区切り）
- 経験レベル: {level}
- トレーニング時間: {time}
- 使用できる器具: {equipment}

【回答形式】
1. ウォームアップ（2〜3種目）
2. メインセット（4〜6種目）
3. クールダウン（1〜2種目）

各種目ごとに「種目名・セット数・回数（または時間）・休憩時間・フォームのポイント」を記載してください。
最後にトレーニング全体のアドバイスも一言添えてください。
```

### 6. app/result/page.tsx（結果表示）

実装要件：

- `"use client"` を先頭に付ける
- `useEffect` で `sessionStorage` から結果（`menu_result`）と条件（`menu_meta`）を取得する
- 結果が存在しない場合は `router.push("/")` でトップに戻す
- 条件のサマリーをインディゴ系のタグ（丸角ピル形式）で表示する
- AI回答は `<pre>` タグで `whitespace-pre-wrap` を使って整形表示する
- 「条件を変えて再生成する」ボタンでトップページに戻れるようにする
- 結果の下に AdSense 広告スペースのプレースホルダーを設置する（破線ボーダー）

---

### 7. app/api/generate/route.ts（APIルート）

実装要件：

- `@google/generative-ai` パッケージを使用する
- `process.env.GEMINI_API_KEY` を使ってクライアントを初期化する
- モデルは `gemini-2.5-flash` を使用する
- リクエストボディから `prompt` を受け取り Gemini に投げる
- 成功時は `{ result: text }` を返す
- エラー時は適切なステータスコードとメッセージを返す

## 構築後に実行してほしいこと

1. 全ファイルを作成する
2. `docker compose build` を実行する
3. `docker compose up` を実行する
4. `http://localhost:3000` でアクセスできることを確認する
5. 問題があれば修正して再度確認する

---

## 注意事項

- `.env.local` は `.env.local.example` をコピーして作成し、
  `GEMINI_API_KEY` には実際のAPIキーを設定すること
- APIキーは `.env.local` のみに記載し、コードにハードコードしないこと
- `node_modules` はコンテナ内で管理し、ホスト側と混在させないこと
- TypeScript の型エラーがないことを確認してから完了とすること
